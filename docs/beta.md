# Adrenalink Beta Version

## Overview

This is the **beta version** of Adrenalink, currently in development for production deployment targeting **2-3 kite schools** within the next **30 days**.

## Beta Phases

### Beta v1 (Initial Release)

- **No Authentication**: Open access app - anyone can use without login
- **Core Functionality**: Basic booking and school discovery features
- **Focus**: Rapid deployment and core feature validation

### Beta v2 (Enhanced Release)

- **Google Authentication**: Secure user login system
- **Middleware**: Protected routes and user session management
- **Enhanced Features**: Full user accounts and personalized experience

## Beta Objectives

The beta version focuses on core functionality that allows:

- **Student Registration**: Users can sign up and create accounts (v2)
- **School Discovery**: Students can search and find appropriate kite schools
- **Booking System**: Create and manage bookings for kite lessons and courses
- **School Management**: Basic functionality for kite schools to manage their offerings

## Target Deployment

- **Timeline**: Production ready within 30 days
- **Target Audience**: 2-3 pilot kite schools
- **Purpose**: Real-world testing and validation before wider release

## Feature Inspiration

This application builds upon the successful functionality from our previous project, the [Kite Hostel](https://github.com/vctrubio/kite-hostel) application, adapting its core booking and management features for kite schools.

## Beta Goals

1. **User Experience Validation**: Test the sign-up and booking flow with real users
2. **School Onboarding**: Validate the school registration and management process
3. **System Stability**: Ensure reliable performance under real-world usage
4. **Feedback Collection**: Gather insights from schools and students for improvements

## Implementation Roadmap

### Beta v1 Implementation

- Complete core booking functionality (no auth required)
- Implement school discovery and listing
- Basic booking form and management
- Deploy to pilot schools for testing

### Beta v2 Implementation

- Integrate Google Authentication
- Add authentication middleware
- Implement user accounts and profiles
- Enhanced booking management with user sessions
- Add payment processing
- Conduct user acceptance testing with pilot schools
- Prepare for full production launch
